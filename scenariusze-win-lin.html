Egzamin INF.02 - Poligon Doświadczalny

Najczęstsze scenariusze współpracy między systemami Windows i Linux

1. Usługi Sieciowe (Web & DNS)

Linux
Klient (Przeglądarka)
HTTP / DNS
Windows
Serwer (IIS + DNS)
  • Windows: Hostuje stronę WWW (IIS) i rozwiązuje nazwy (Strefa DNS).
  • Linux: Wpisuje w przeglądarkę np. www.egzamin.local.
🔥 Haczyk egzaminacyjny: Jeśli strona się nie ładuje, sprawdź czy Zapora Windows przepuszcza port 80 oraz czy Linux ma ustawiony adres IP Windowsa jako swój serwer DNS!

2. Windows jako Router (NAT)

Linux
Host w sieci LAN
Ruch do WAN
Windows
Brama (RRAS / NAT)
  • Windows: Ma 2 karty sieciowe. Działa jako router z włączonym NAT.
  • Linux: Ma ustawiony adres IP Windowsa jako swoją Bramę Domyślną.
🔥 Haczyk egzaminacyjny: Linux musi umieć spingować adres w sieci zewnętrznej (WAN). Bez poprawnego NAT na Windowsie, pakiety nie wrócą do Linuxa.

3. Wirtualizacja (VirtualBox)

Windows
Gospodarz (Host)
Mostek (Bridged)
Linux
Maszyna Wirtualna
  • Windows: Udostępnia zasoby sprzętowe przez VirtualBox.
  • Linux: Działa w oknie, ale musi być widoczny w fizycznej sieci.
🔥 Haczyk egzaminacyjny: Domyślnie VirtualBox ustawia sieć na "NAT". Musisz zmienić to na "Karta Mostkowana" (Bridged), aby Linux dostał IP z tej samej podsieci co fizyczny Windows!

4. Diagnostyka i Zapora (Ping)

Linux
Wysyła Ping
ICMP Echo
Windows
Odbiera Ping
  • Oba systemy: Muszą się wzajemnie "widzieć" w sieci (ping).
🔥 Haczyk egzaminacyjny: Linux domyślnie odpowie na ping od Windowsa. Ale Windows domyślnie zablokuje ping od Linuxa! Musisz wejść w Zaporę Windows i włączyć regułę dla "Udostępnianie plików i drukarek (żądanie echa - ICMPv4-In)".
