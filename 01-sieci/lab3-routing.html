Lab 3: DHCP, DNS, Routing i NAT

Faza 1: Serwer DHCP (Tryb Graficzny)

⏱️ Czas: 45 minut
Cel: Skonfigurowanie dedykowanego serwera, aby automatycznie rozdawał adresy IP komputerom w dziale IT (VLAN 10). Koniec z ręcznym wpisywaniem!
  • 🖧 Switch 1
    • 👨‍💻 PC1 (IT)
    • F0/3 🗄️ Serwer DHCP 192.168.10.2
  1. Baza: Używamy topologii z Lab 2. Dodaj urządzenie Server-PT i podłącz je kablem do portu FastEthernet0/3 na Switchu (ten port musi być w VLAN 10!).
  2. Adres IP Serwera: Serwer musi mieć stały adres, żeby komputery mogły go znaleźć.
    🖱️ Kliknij Serwer ➔ Zakładka Desktop ➔ IP Configuration
    Ustaw: IP 192.168.10.2, Maska 255.255.255.0, Default Gateway 192.168.10.1.
  3. Konfiguracja usługi DHCP: Włączamy rozdawanie adresów.
    🖱️ Kliknij Serwer ➔ Zakładka Services ➔ Po lewej wybierz DHCP
    • Zaznacz Service: On
    • Default Gateway: wpisz 192.168.10.1
    • Start IP Address: zmień końcówkę na 100 (czyli 192.168.10.100)
    • Kliknij przycisk Save (zapisz pulę).
  4. Testowanie: Wejdź na PC1 (Dział IT).
    🖱️ Kliknij PC1 ➔ Zakładka Desktop ➔ IP Configuration
    Zmień kropkę ze Static na DHCP. Komputer powinien błyskawicznie pobrać adres 192.168.10.100!
🤖
Wniosek: Serwer DHCP to "urzędnik", który automatycznie przydziela adresy zamieszkania (IP) nowym komputerom w sieci. Dzięki temu administrator nie musi biegać od komputera do komputera!

Faza 2: Wyjście w Świat (Routing w GUI)

⏱️ Czas: 1 godzina 15 minut
Cel: Podłączenie naszej firmy do Dostawcy Internetu (ISP) i wyklikanie tras tak, aby pakiety trafiały do serwera Google (8.8.8.8) i WRACAŁY.
  • G0/0: 200.0.0.2 🏢 Router ISP (Dostawca)
    • G0/1: 200.0.0.1 🌐 Nasz Router (Firma)
    • G0/1: 8.8.8.1 🌍 Serwer Google 8.8.8.8
  1. Zadanie: Dodaj nowy Router (nazwij go ISP) oraz Serwer (nazwij go Google). Połącz Nasz Router (port G0/1) z Routerem ISP (port G0/0). Połącz Router ISP (port G0/1) z Serwerem Google.
  2. Włączanie portów i adresacja (Nasz Router):
    🖱️ Kliknij Nasz Router ➔ Zakładka Config ➔ Po lewej GigabitEthernet0/1
    Zaznacz Port Status: On. Wpisz IP: 200.0.0.1, Maska: 255.255.255.252.
  3. Włączanie portów i adresacja (Router ISP):
    🖱️ Kliknij Router ISP ➔ Zakładka Config
    • Dla GigabitEthernet0/0: Port Status On, IP: 200.0.0.2, Maska: 255.255.255.252.
    • Dla GigabitEthernet0/1: Port Status On, IP: 8.8.8.1, Maska: 255.255.255.0.
  4. Konfiguracja Serwera Google: Wejdź w jego zakładkę Desktop -> IP Config. Ustaw IP: 8.8.8.8, Maska 255.255.255.0, Brama 8.8.8.1.
  5. Routing Domyślny (Nasz Router): Mówimy routerowi: "Wszystko, czego nie znasz, wysyłaj do ISP".
    🖱️ Kliknij Nasz Router ➔ Zakładka Config ➔ Po lewej sekcja Routing: Static
    Wpisz Network: 0.0.0.0, Mask: 0.0.0.0, Next Hop: 200.0.0.2. Kliknij przycisk Add.
  6. Routing Powrotny (ISP): Uczymy dostawcę drogi powrotnej do naszej firmy (do VLAN 10).
    🖱️ Kliknij Router ISP ➔ Zakładka Config ➔ Po lewej sekcja Routing: Static
    Wpisz Network: 192.168.10.0, Mask: 255.255.255.0, Next Hop: 200.0.0.1. Kliknij przycisk Add.
  7. Wielki Test: Wejdź na PC1. Otwórz Command Prompt i wpisz ping 8.8.8.8. Działa! Właśnie podłączyłeś firmę do Internetu.
🛣️
Wniosek: Routing to ulica dwukierunkowa! Trasa domyślna (0.0.0.0) to genialne narzędzie – wypycha cały nieznany ruch do dostawcy. Ale pamiętaj: dostawca (ISP) musi mieć wpisane trasy statyczne, aby wiedzieć, jak odesłać pakiety z powrotem do Twojej sieci lokalnej.

Faza 3: DNS (Książka Telefoniczna)

⏱️ Czas: 30 minut
Cel: Sprawić, by użytkownicy mogli wpisywać w przeglądarce www.google.com zamiast trudnego do zapamiętania adresu IP 8.8.8.8.
  1. Baza: Rozbudowujemy topologię z Fazy 2. Użyjemy Serwera Google (8.8.8.8) również jako serwera DNS.
  2. Konfiguracja usługi DNS: Uczymy serwer, jak tłumaczyć nazwy na IP.
    🖱️ Kliknij Serwer Google ➔ Zakładka Services ➔ Po lewej DNS
    • Zaznacz DNS Service: On
    • W polu Name wpisz: www.google.com
    • W polu Address wpisz: 8.8.8.8
    • Kliknij przycisk Add.
  3. Aktualizacja DHCP: Komputery w naszej firmie muszą wiedzieć, kogo pytać o nazwy stron!
    🖱️ Kliknij Serwer DHCP (w naszej firmie) ➔ Zakładka Services ➔ DHCP
    W polu DNS Server wpisz 8.8.8.8 i kliknij Save.
  4. Odświeżenie IP na PC: Wejdź na PC1, przełącz IP na Static i z powrotem na DHCP. Sprawdź, czy w polu DNS Server pojawiło się 8.8.8.8.
  5. Wielki Test: Na PC1 otwórz zakładkę Desktop, wybierz Web Browser (Przeglądarka). W pasku adresu wpisz www.google.com i wciśnij Enter. Powinna załadować się strona!
📖
Wniosek: DNS to książka telefoniczna internetu. Komputery komunikują się TYLKO za pomocą adresów IP. Kiedy wpisujesz nazwę strony, Twój komputer najpierw pyta serwera DNS: "Jaki IP ma www.google.com?", a dopiero potem wysyła tam właściwe dane.

Faza 4: NAT (Magiczna Peleryna)

⏱️ Czas: 1 godzina
Cel: Ukrycie całej naszej prywatnej sieci (192.168.x.x) pod jednym publicznym adresem IP. Tak działa każdy domowy router!
  • 🏢 Internet (ISP) Odrzuca prywatne IP!
    • 🎭 Nasz Router (NAT) Tłumaczy 192.168.x.x na 200.0.0.1
      • 🖧 Nasza Sieć Prywatna
  1. Symulacja prawdziwego świata: W Fazie 2 dodaliśmy na routerze ISP trasę powrotną do naszej sieci. W prawdziwym świecie ISP tego NIE ZROBI (adresy 192.168.x.x są zablokowane w internecie).
    🖱️ Kliknij Router ISP ➔ Zakładka Config ➔ Static
    Zaznacz trasę 192.168.10.0 i kliknij Remove. Zrób to samo dla 192.168.20.0.
    Spróbuj teraz spingować 8.8.8.8 z PC1. Przestało działać! Pakiet tam dociera, ale nie potrafi wrócić.
  2. Konfiguracja NAT (CLI): Musimy włączyć NAT na Naszym Routerze. Zrobimy to w CLI, bo to zadanie dla prawdziwych inżynierów. Najpierw określamy, który port jest "wewnątrz" firmy, a który "na zewnątrz".
! Na Naszym Routerze (Firma): enable configure terminal ! Port w stronę naszej firmy (Inside) interface gigabitEthernet 0/0 ip nat inside exit ! Port w stronę Internetu (Outside) interface gigabitEthernet 0/1 ip nat outside exit
  1. Lista dostępu (ACL): Mówimy routerowi, komu pozwalamy na wyjście do internetu. Pozwolimy wszystkim z naszej sieci.
! Tworzymy listę nr 1, która pozwala na wszystko access-list 1 permit any
  1. Uruchomienie magii (PAT / NAT Overload): Łączymy listę z portem wyjściowym.
! Tłumacz adresy z listy 1 na publiczny adres portu G0/1 ip nat inside source list 1 interface gigabitEthernet 0/1 overload
  1. Wielki Test: Wejdź na PC1 i ponownie spinguj 8.8.8.8. Działa!
  2. Zajrzyj pod maskę: Na Naszym Routerze w CLI wpisz komendę:
    show ip nat translations. Zobaczysz tabelkę, w której router zapisuje, jak podmienił prywatny adres PC1 na swój publiczny adres 200.0.0.1!
🎭
Wniosek: NAT (Network Address Translation) to tłumacz. Kiedy PC1 wysyła pakiet do Google, Nasz Router zdejmuje z niego prywatny adres (192.168.10.100), nakleja swój publiczny (200.0.0.1) i wysyła w świat. Google odpowiada do routera, a router pamięta, kto o to pytał, i przekazuje odpowiedź do PC1. Dzięki temu cała szkoła/firma może korzystać z internetu, mając wykupiony tylko JEDEN publiczny adres IP!
