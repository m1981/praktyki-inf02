Ćwiczenie 3: Automatyzacja i Wyjście w Świat

Faza 1: Serwer DHCP (Tryb Graficzny)

⏱️ Czas: 45 minut
Cel: Skonfigurowanie dedykowanego serwera, aby automatycznie rozdawał adresy IP komputerom w dziale IT (VLAN 10). Koniec z ręcznym wpisywaniem!
  • 🖧 Switch 1
    • 👨‍💻 PC1 (IT)
    • F0/3 🗄️ Serwer DHCP 192.168.10.2
  1. Baza: Używamy topologii z Lab 2. Dodaj urządzenie Server-PT i podłącz je kablem do portu FastEthernet0/3 na Switchu (ten port musi być w VLAN 10!).
  2. Adres IP Serwera: Serwer musi mieć stały adres, żeby komputery mogły go znaleźć.
    🖱️ Kliknij Serwer ➔ Zakładka Desktop ➔ IP Configuration
    Ustaw: IP 192.168.10.2, Maska 255.255.255.0, Default Gateway 192.168.10.1.
  3. Konfiguracja usługi DHCP: Włączamy rozdawanie adresów.
    🖱️ Kliknij Serwer ➔ Zakładka Services ➔ Po lewej wybierz DHCP
    • Zaznacz Service: On
    • Default Gateway: wpisz 192.168.10.1
    • Start IP Address: zmień końcówkę na 100 (czyli 192.168.10.100)
    • Kliknij przycisk Save (zapisz pulę).
  4. Testowanie: Wejdź na PC1 (Dział IT).
    🖱️ Kliknij PC1 ➔ Zakładka Desktop ➔ IP Configuration
    Zmień kropkę ze Static na DHCP. Komputer powinien błyskawicznie pobrać adres 192.168.10.100!
🤖
Wniosek: Serwer DHCP to "urzędnik", który automatycznie przydziela adresy zamieszkania (IP) nowym komputerom w sieci. Dzięki temu administrator nie musi biegać od komputera do komputera!

Faza 2: Wyjście w Świat (Routing w GUI)

⏱️ Czas: 1 godzina 15 minut
Cel: Podłączenie naszej firmy do Dostawcy Internetu (ISP) i wyklikanie tras tak, aby pakiety trafiały do serwera Google (8.8.8.8) i WRACAŁY.
  • G0/0: 200.0.0.2 🏢 Router ISP (Dostawca)
    • G0/1: 200.0.0.1 🌐 Nasz Router (Firma)
    • G0/1: 8.8.8.1 🌍 Serwer Google 8.8.8.8
  1. Zadanie: Dodaj nowy Router (nazwij go ISP) oraz Serwer (nazwij go Google). Połącz Nasz Router (port G0/1) z Routerem ISP (port G0/0). Połącz Router ISP (port G0/1) z Serwerem Google.
  2. Włączanie portów i adresacja (Nasz Router):
    🖱️ Kliknij Nasz Router ➔ Zakładka Config ➔ Po lewej GigabitEthernet0/1
    Zaznacz Port Status: On. Wpisz IP: 200.0.0.1, Maska: 255.255.255.252.
  3. Włączanie portów i adresacja (Router ISP):
    🖱️ Kliknij Router ISP ➔ Zakładka Config
    • Dla GigabitEthernet0/0: Port Status On, IP: 200.0.0.2, Maska: 255.255.255.252.
    • Dla GigabitEthernet0/1: Port Status On, IP: 8.8.8.1, Maska: 255.255.255.0.
  4. Konfiguracja Serwera Google: Wejdź w jego zakładkę Desktop -> IP Config. Ustaw IP: 8.8.8.8, Maska 255.255.255.0, Brama 8.8.8.1.
  5. Routing Domyślny (Nasz Router): Mówimy routerowi: "Wszystko, czego nie znasz, wysyłaj do ISP".
    🖱️ Kliknij Nasz Router ➔ Zakładka Config ➔ Po lewej sekcja Routing: Static
    Wpisz Network: 0.0.0.0, Mask: 0.0.0.0, Next Hop: 200.0.0.2. Kliknij przycisk Add.
  6. Routing Powrotny (ISP): Uczymy dostawcę drogi powrotnej do naszej firmy (do VLAN 10).
    🖱️ Kliknij Router ISP ➔ Zakładka Config ➔ Po lewej sekcja Routing: Static
    Wpisz Network: 192.168.10.0, Mask: 255.255.255.0, Next Hop: 200.0.0.1. Kliknij przycisk Add.
  7. Wielki Test: Wejdź na PC1. Otwórz Command Prompt i wpisz ping 8.8.8.8. Działa! Właśnie podłączyłeś firmę do Internetu.
🛣️
Wniosek: Routing to ulica dwukierunkowa! Trasa domyślna (0.0.0.0) to genialne narzędzie – wypycha cały nieznany ruch do dostawcy. Ale pamiętaj: dostawca (ISP) musi mieć wpisane trasy statyczne, aby wiedzieć, jak odesłać pakiety z powrotem do Twojej sieci lokalnej.
