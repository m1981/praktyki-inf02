🔗 Filar 6: Integracja Win-Lin

Kiedy dwa różne światy muszą zacząć ze sobą rozmawiać

Krok 1

Po co łączyć ogień z wodą?

🌍 Prawdziwe IT
  • Mit: W firmach używa się tylko Windowsa albo tylko Linuxa.
  • Fakt: Każdy system ma swoje mocne strony. Windows jest genialny do zarządzania ludźmi (Active Directory) i stacjami roboczymi. Linux jest niepokonany jako lekki serwer WWW, baza danych czy narzędzie sieciowe.
  • Na egzaminie INF.02: Egzaminatorzy symulują to środowisko. Zazwyczaj Windows to Serwer (dostarcza usługi), a Linux to Klient (konsumuje usługi i służy do testowania).
💡 Analogia: Wyobraź sobie restaurację. Windows to "Sala dla gości i Menadżer" (ładny wygląd, zarządzanie kelnerami, Active Directory). Linux to "Kuchnia" (tam się ciężko pracuje, jest gorąco, nie ma ładnych okienek, ale powstają tam najlepsze rzeczy). Restauracja potrzebuje obu tych działów, żeby działać!
Krok 2

Wspólny język (Protokoły)

🗣️ Komunikacja
  • Systemy się nie widzą: Linux nie wie, że rozmawia z Windowsem. Windows nie wie, że rozmawia z Linuxem.
  • Liczy się standard: Komunikacja odbywa się przez uniwersalne protokoły. Jeśli Windows wystawi stronę po HTTP (port 80), to przeglądarka na Linuxie ją zrozumie.
  • Kluczowe protokoły na egzaminie: ICMP (Ping), DNS (rozwiązywanie nazw), HTTP (strony WWW), DHCP (przydzielanie IP).
💡 Analogia: To tak, jakby spotkał się Polak z Japończykiem. Nie znają swoich języków ojczystych (Windows/Linux), ale obaj świetnie mówią po angielsku (TCP/IP). Dzięki temu mogą bez problemu ubić interes.
Krok 3

Incepcja: System w Systemie

📦 Wirtualizacja
  • VirtualBox: Na nowszych arkuszach egzaminacyjnych nie dostaniesz dwóch fizycznych komputerów. Dostaniesz Windowsa, na którym musisz postawić Linuxa w maszynie wirtualnej.
  • Karta Mostkowana (Bridged): To najważniejsze ustawienie! Domyślnie maszyna wirtualna jest odcięta od sieci fizycznej (tryb NAT). Musisz zrobić "most", żeby Linux dostał IP z tej samej puli co fizyczny Windows.
💡 Analogia: Tryb NAT to jak mieszkanie w hotelu – masz internet, ale nikt z zewnątrz nie wie, w którym jesteś pokoju. Tryb Bridged (Mostkowana) to jak wybudowanie własnego domu na tej samej ulicy co hotel – dostajesz własny, widoczny dla wszystkich adres.

🕵️‍♂️ Sekret Inżyniera: Asymetria Zapory (Firewall)

🚨 Najczęstszy błąd na egzaminie!

Wyobraź sobie, że połączyłeś kable, ustawiłeś adresy IP i chcesz sprawdzić, czy komputery się widzą. Używasz komendy ping.

  • Windows ➔ Linux: Działa od razu! Linux z natury jest ufny i odpowiada na pingi.
  • Linux ➔ Windows: Request Timeout (Upłynął limit czasu żądania). Panika! Kable zepsute? Złe IP?
💡 Pamiętaj: Windows domyślnie BLOKUJE wszystkie zapytania z zewnątrz, w tym Ping (ICMP). Zanim zaczniesz szukać błędu w kablach, wejdź w Zaporę Windows (wf.msc) i włącz regułę przychodzącą dla "Żądania echa (ICMPv4-In)". Bez tego oblejesz test komunikacji!
🎓
Podsumowanie dla Was: Ten moduł to ostateczny test Waszych umiejętności. Jeśli potraficie sprawić, że Linux pobierze stronę WWW z Windowsa, a Windows wypuści Linuxa do Internetu przez swój routing – jesteście gotowi na egzamin i prawdziwą pracę w IT!
