LAB 1: "Gdzie jest moja strona?"

Cel: Udowodnić, że Linux potrafi korzystać z usług Web i DNS hostowanych na Windows Server.
Windows Server 2022
Serwer (IIS + DNS)
ZAPYTANIE HTTP/DNS
Ubuntu Desktop
Klient (Przeglądarka)

Krok 1: Instalacja Ról

W Menedżerze Serwera zainstaluj dwie role: Serwer DNS oraz Serwer sieci Web (IIS).

Krok 2: Konfiguracja DNS

Otwórz przystawkę DNS. Utwórz nową Strefę wyszukiwania do przodu o nazwie egzamin.local. Wewnątrz strefy dodaj nowy Rekord A (Host) o nazwie www, wskazujący na adres IP tego Windows Servera.

Krok 3: Konfiguracja IIS (Web Server)

Utwórz folder C:\www i stwórz w nim plik index.html z dowolnym tekstem. Otwórz Menedżer IIS, dodaj nową witrynę wskazującą na ten folder. W oknie Wiązania (Bindings) w polu Nazwa hosta wpisz koniecznie: www.egzamin.local.

Krok 4: Konfiguracja Sieci Klienta

Na Ubuntu wejdź w ustawienia karty sieciowej. Zmień adres Serwera DNS na adres IP Twojego Windows Servera. Zapisz i zrestartuj połączenie sieciowe.

Krok 5: Wielki Test

Otwórz przeglądarkę Firefox na Ubuntu i wpisz w pasek adresu: http://www.egzamin.local. Strona powinna się załadować!

HACZYKI EGZAMINACYJNE (Arkusze 24.01, 25.01)
  • Zapora (Firewall): Strona się nie ładuje (Timeout)? Windows domyślnie blokuje ruch z zewnątrz! Musisz wejść w wf.msc na Windowsie i utworzyć regułę przychodzącą zezwalającą na ruch na porcie 80 (TCP).
  • Uprawnienia NTFS: Widzisz błąd 401/403 (Brak dostępu) w przeglądarce? Sprawdź, czy folder C:\www ma w zakładce Zabezpieczenia dodane uprawnienia "Odczyt" dla grupy Użytkownicy (lub IIS_IUSRS).
Pro-Tip od DevOpsa: Zamiast klikać w przeglądarkę, otwórz terminal na Ubuntu i wpisz curl -v http://www.egzamin.local. Zobaczysz dokładnie, czy problem leży w DNS (nie potrafi rozwiązać nazwy), czy w zaporze (brak połączenia), czy w uprawnieniach (kod HTTP 403).
