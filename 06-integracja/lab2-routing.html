Routing i Translacja Adresów

Czas: 45 min
Cel: Skonfigurować usługę RRAS (NAT), aby Linux mógł połączyć się z Internetem przez Windows Server.
  • Krok 1: Dwie karty sieciowe (Multihoming)

    Upewnij się, że Windows Server ma dwie karty sieciowe. Karta 1 (WAN) ma dostęp do Internetu. Karta 2 (LAN) musi mieć ustawiony statyczny adres IP, np. 10.0.0.1. Ważne: Karta LAN nie może mieć ustawionej bramy domyślnej!

  • Krok 2: Instalacja Roli RRAS

    W Menedżerze Serwera dodaj rolę: Dostęp zdalny (Remote Access), a w jej szczegółach zaznacz Routing.

  • Krok 3: Konfiguracja NAT

    Otwórz przystawkę Routing i dostęp zdalny (rrasmgmt.msc). Kliknij prawym przyciskiem myszy na nazwę serwera i wybierz Konfiguruj i włącz.... W kreatorze wybierz Translacja adresów sieciowych (NAT). Następnie wskaż Kartę 1 (WAN) jako interfejs publiczny podłączony do Internetu.

  • Krok 4: Konfiguracja Klienta (Linux)

    Na Ubuntu ustaw statyczny adres IP z tej samej podsieci, np. 10.0.0.2. Jako Bramę domyślną (Gateway) wpisz adres LAN Windowsa: 10.0.0.1. Jako DNS ustaw np. 8.8.8.8.

  • Krok 5: Wielki Test (Ping w świat)

    Otwórz terminal na Ubuntu i wpisz: ping 8.8.8.8. Jeśli pakiety wracają, oznacza to, że Windows Server poprawnie tłumaczy adresy (NAT) i wypuszcza Linuxa w świat!

🚨 HACZYKI EGZAMINACYJNE (Arkusz 24.01):
  • Dwie Bramy Domyślne: To najczęstszy błąd! Windows Server nie może mieć wpisanej bramy domyślnej na karcie LAN. Brama domyślna może być tylko jedna w całym systemie (na karcie WAN). Jeśli wpiszesz dwie, tablica routingu zwariuje i NAT nie zadziała.
  • Brak Bramy na Linuxie: Jeśli zapomnisz wpisać 10.0.0.1 jako bramę w ustawieniach Ubuntu, Linux nie będzie wiedział, komu przekazać pakiety przeznaczone do Internetu. Zwróci błąd Network is unreachable.
  • Zły interfejs w kreatorze RRAS: Kreator pyta o interfejs publiczny (podłączony do Internetu). Jeśli przez pomyłkę wskażesz tam kartę LAN, zablokujesz cały ruch.
💡 Pro-Tip od DevOpsa: Chcesz zobaczyć magię routingu na własne oczy? Na Linuxie wpisz komendę traceroute 8.8.8.8 (lub tracepath 8.8.8.8). Zobaczysz, że pierwszym "przystankiem" (hop) Twoich pakietów jest adres IP Windows Servera (10.0.0.1), a dopiero potem lecą one do dostawcy Internetu.
