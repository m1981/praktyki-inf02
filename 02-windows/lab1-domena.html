💻 Laboratoria: Budujemy Szkolną Sieć

Praktyka czyni mistrza (Zadania z arkuszy INF.02)

Lab 1: Narodziny Szefa (Kontroler Domeny)

Czas: 45 min
Cel: Przekształcenie zwykłego Windows Servera w Kontroler Domeny i podłączenie do niego stacji roboczej ucznia. (Z arkuszy: v1.1, v1.7)
  • Krok 1: Instalacja roli AD DS
    Server Manager (Menedżer serwera) ➔ Add roles and features (Dodaj role i funkcje) ➔ Active Directory Domain Services (Usługi domenowe Active Directory)
  • Krok 2: Promowanie serwera

    Po instalacji kliknij żółty trójkąt ostrzegawczy w Menedżerze serwera i wybierz "Promote this server to a domain controller" (Podnieś poziom tego serwera do poziomu kontrolera domeny).

    • Wybierz: Add a new forest (Dodaj nowy las).
    • Root domain name (Nazwa domeny głównej): zs1.local
    • Ustaw hasło przywracania usług katalogowych (DSRM) na Zaq12345!
    • 🚨 Najczęstszy błąd uczniów: Próba podłączenia Windowsa 10 do domeny kończy się błędem "Nie można odnaleźć domeny". Dzieje się tak, ponieważ stacja robocza nie wie, gdzie szukać domeny zs1.local.
    💡 Złota Rada Trenera: Zanim klikniesz "Dołącz do domeny", wejdź w ustawienia karty sieciowej Windows 10 (ncpa.cpl) i w polu Preferowany serwer DNS wpisz adres IP Twojego Windows Servera! Następnie otwórz CMD i wpisz ping zs1.local. Jeśli odpowiada – możesz dołączać!
  • Krok 3: Podłączenie stacji roboczej (Windows 10)

    Uruchom drugą maszynę wirtualną (Windows 10). Upewnij się, że w ustawieniach karty sieciowej ma wpisany adres IP serwera jako swój serwer DNS!

    System Properties (Właściwości systemu) ➔ Change settings (Zmień ustawienia) ➔ Domain (Domena): zs1.local

    Podaj login i hasło Administratora domeny, aby autoryzować podłączenie.

📸 Weryfikacja Egzaminacyjna (Jak zdobyć punkty?)

Na egzaminie INF.02 nikt nie uwierzy Ci na słowo. Musisz to udowodnić:

  1. Na stacji roboczej (Windows 10) otwórz okno Właściwości systemu (tam, gdzie widać pełną nazwę komputera i nazwę domeny).
  2. Zrób zrzut ekranu (PrintScreen lub Narzędzie Wycinanie).
  3. Otwórz edytor tekstu (Word / LibreOffice Writer), wklej zrzut ekranu.
  4. Zapisz plik pod nazwą np. stacja_testy.docx na pendrive opisanym jako EGZAMIN-X (gdzie X to numer Twojego stanowiska).

Lab 2: Władza z Klawiatury (Zarządzanie AD)

Czas: 45 min
Cel: Zbudowanie struktury szkoły w Active Directory używając zarówno myszki (GUI), jak i wiersza poleceń (CLI). (Z arkuszy: v1.1, v1.3, v1.4)
  • Krok 1: Struktura (GUI)
    Narzędzia ➔ Użytkownicy i komputery usługi Active Directory (dsa.msc)

    Utwórz nową Jednostkę Organizacyjną (OU) o nazwie Pracownia_32. Wewnątrz niej utwórz użytkownika maciek. We właściwościach konta zaznacz opcję "Hasło nigdy nie wygasa".

    • 🚨 Najczęstszy błąd uczniów: Wpisywanie złego formatu daty w komendzie /expires. Zależnie od języka systemu, Windows może oczekiwać formatu DD/MM/RRRR lub MM/DD/RRRR. Jeśli komenda wyrzuca błąd, zamień dzień z miesiącem!
  • Krok 2: Magia CLI (Wiersz poleceń)

    Egzaminatorzy uwielbiają sprawdzać znajomość konsoli. Otwórz CMD jako Administrator i wykonaj poniższe polecenia:

    # 1. Utworzenie grupy lokalnej "Uczniowie" net localgroup Uczniowie /add # 2. Utworzenie użytkownika z datą wygaśnięcia konta net user uczen1 Zaq12345! /add /expires:25/06/2025 # 3. Dodanie użytkownika do grupy net localgroup Uczniowie uczen1 /add

📝 Weryfikacja Egzaminacyjna (Papierowa!)

W przypadku zadań z wiersza poleceń (CLI), egzaminator nie zawsze prosi o zrzut ekranu. W arkuszu egzaminacyjnym znajduje się fizyczna tabela (np. Tabela 2. Polecenia Windows). Musisz w niej długopisem wpisać dokładną komendę, której użyłeś, wraz ze wszystkimi parametrami (np. net user uczen1 haslo /add).

Lab 3: Tajne Akta i Wędrujące Pulpity

Czas: 60 min
Cel: Konfiguracja uprawnień do plików (NTFS), ukrytych udziałów sieciowych oraz profili mobilnych. (Z arkuszy: v1.2, v1.4, v1.7)
  • Krok 1: Ukryty udział dla profili

    Utwórz na dysku C: folder Profile. Udostępnij go w sieci, ale dodaj znak $ na końcu nazwy udziału (Profile$). Dzięki temu będzie niewidoczny dla wścibskich uczniów.

  • Krok 2: Profil Mobilny (Roaming Profile)
    Właściwości użytkownika 'maciek' w AD ➔ Zakładka: Profil ➔ Ścieżka profilu

    Wpisz ścieżkę UNC: \\NAZWA_SERWERA\Profile$\%username%. Zaloguj się jako maciek na Windows 10, stwórz plik na pulpicie, wyloguj się i sprawdź na serwerze, czy plik się zsynchronizował!

    • 💡 Złota Rada Trenera (Najbardziej restrykcyjne wygrywa): Jeśli na zakładce Udostępnianie dasz "Pełną kontrolę", ale w Zabezpieczeniach (NTFS) dasz tylko "Odczyt", to użytkownik będzie miał tylko odczyt. Zawsze dawaj "Wszyscy - Pełna kontrola" w Udostępnianiu, a prawdziwe restrykcje ustawiaj w Zabezpieczeniach!
  • Krok 3: Uprawnienia NTFS i Szyfrowanie (EFS)

    Utwórz folder Sprawdziany. W zakładce Zabezpieczenia (NTFS) ustaw:

    • Administratorzy: Pełna kontrola
    • Użytkownicy: Odczyt i wykonywanie

    Zaloguj się jako uczeń, stwórz plik tekstowy i zaszyfruj go (Właściwości pliku -> Zaawansowane -> Szyfruj zawartość). Spróbuj go otworzyć z konta Administratora (Odmowa dostępu!).

    • 🚨 Najczęstszy błąd uczniów (EFS): Uczeń szyfruje plik będąc zalogowanym jako Administrator, a potem dziwi się, że zwykły użytkownik nie może go otworzyć. Pamiętaj: EFS szyfruje plik dla osoby, która aktualnie klika "Szyfruj"!

📸 Weryfikacja Egzaminacyjna (Zrzuty ekranu)

Aby udowodnić poprawne uprawnienia i szyfrowanie:

  1. Zrób zrzut ekranu zakładki Zabezpieczenia folderu Sprawdziany, pokazując uprawnienia dla grupy Użytkownicy.
  2. Zrób zrzut ekranu okna Eksploratora, na którym widać, że nazwa zaszyfrowanego pliku zmieniła kolor na zielony (lub zrób zrzut okna "Atrybuty zaawansowane" z zaznaczonym ptaszkiem przy szyfrowaniu).

Lab 4: Szkolny Intranet (DNS i IIS)

Czas: 60 min
Cel: Uruchomienie szkolnej strony WWW i konfiguracja serwera nazw (DNS), aby strona odpowiadała na przyjazny adres. (Z arkuszy: v1.2, v1.5, v1.6, v1.8)
  • Krok 1: Instalacja ról

    Zainstaluj role: Serwer DNS oraz Serwer sieci Web (IIS).

  • Krok 2: Konfiguracja DNS
    Menedżer DNS ➔ Strefy wyszukiwania do przodu ➔ Nowa strefa

    Utwórz strefę intranet.local. Wejdź w jej właściwości (zakładka SOA) i zmień Interwał odświeżania na 10 minut.

    Utwórz Rekord A (nazwa: www, adres IP: adres Twojego serwera).

    • 💡 Złota Rada Trenera (Testuj DNS przed IIS!): Zanim zaczniesz konfigurować stronę WWW, otwórz CMD na stacji roboczej i wpisz ping www.intranet.local. Jeśli ping nie rozwiązuje nazwy na IP serwera, to strona na pewno nie zadziała. Zawsze najpierw naprawiaj DNS!
  • Krok 3: Konfiguracja IIS (Host Headers)

    Utwórz prosty plik index.html z napisem "Witaj w szkole" i wrzuć go do folderu C:\inetpub\wwwroot\szkola.

    Menedżer IIS ➔ Witryny ➔ Dodaj witrynę sieci Web

    Wskaż ścieżkę fizyczną. W sekcji Wiązania (Bindings) wpisz w polu Nazwa hosta: www.intranet.local.

🌍
Sprawdź się: Otwórz przeglądarkę na stacji roboczej Windows 10 i wpisz http://www.intranet.local. Jeśli widzisz swoją stronę, to znaczy, że DNS poprawnie przetłumaczył nazwę na IP, a IIS zaserwował odpowiedni plik!
🚨 Najczęstszy błąd uczniów (Błąd 403 w IIS): Jeśli usuniesz domyślne uprawnienia z folderu C:\inetpub\wwwroot\szkola, serwer IIS nie będzie mógł odczytać pliku index.html. IIS używa specjalnej grupy IIS_IUSRS do czytania plików. Nie usuwaj jej!

🙋‍♂️ Weryfikacja Egzaminacyjna (Podniesienie ręki)

W zadaniach z IIS i DNS egzaminator zazwyczaj chce zobaczyć efekt końcowy na żywo:

  1. Otwórz przeglądarkę na stacji roboczej (Windows 10 lub wirtualny Linux).
  2. Wpisz w pasku adresu: http://www.intranet.local.
  3. Gdy strona się załaduje, podnieś rękę. Egzaminator podejdzie i sprawdzi, czy strona ładuje się po nazwie domenowej, a nie po samym adresie IP.
