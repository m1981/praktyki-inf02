🐧 Lab 4: Gdzie jest Internet?

Konfiguracja interfejsów sieciowych, adresacji IP i Netplan

Cel: Nauczyć się diagnozować połączenie sieciowe, zmieniać nazwę hosta oraz konfigurować statyczny adres IP w nowoczesnym systemie Ubuntu.

Krok 1: Rozpoznanie (Recon)

Czas: 5 min

Zanim cokolwiek zmienisz, musisz wiedzieć, jak nazywa się Twoja karta sieciowa i jaki ma obecnie adres IP.

# 1. Wyświetl wszystkie karty sieciowe i ich adresy IP ip a # 2. Sprawdź tablicę routingu (gdzie jest brama domyślna / default gateway?) ip route # 3. Sprawdź, czy masz wyjście na świat (przerwij wciskając Ctrl+C!) ping 8.8.8.8
💡 Złota Rada Trenera (Nazwy interfejsów): W starych Linuxach karta sieciowa zawsze nazywała się eth0. W nowoczesnym Ubuntu nazwy są generowane na podstawie portu na płycie głównej, np. enp0s3 lub ens33. Zawsze najpierw wpisz ip a, aby poznać dokładną nazwę swojej karty!

Krok 2: Zmiana nazwy komputera (Hostname)

Czas: 5 min

To jedno z najczęstszych zadań na egzaminie. Egzaminator prosi o zmianę nazwy stacji roboczej.

# 1. Sprawdź obecną nazwę komputera hostname # 2. Zmień nazwę na STACJAX (wymaga uprawnień root!) sudo hostnamectl set-hostname STACJAX # 3. Wyloguj się i zaloguj ponownie, aby zobaczyć zmianę w znaku zachęty exit
Zobacz oryginalne zadanie z egzaminu

Egzamin: Styczeń 2022 (INF.02-01-22.01-SG)

Treść zadania:
‒ zmień nazwę komputera na STACJAX, gdzie X to numer Twojego stanowiska egzaminacyjnego

Krok 3: Konfiguracja IP (Netplan) - BOSS FIGHT 🐉

Czas: 20 min

W Ubuntu od wersji 18.04 sieć konfiguruje się za pomocą narzędzia Netplan. Używa ono plików w formacie YAML. Format ten jest niezwykle wrażliwy na spacje.

# 1. Znajdź plik konfiguracyjny (zazwyczaj nazywa się 00-installer-config.yaml lub 50-cloud-init.yaml) ls /etc/netplan/ # 2. Edytuj ten plik w nano (podmień nazwę pliku na tę, którą znalazłeś wyżej!) sudo nano /etc/netplan/50-cloud-init.yaml

W edytorze nano zmień konfigurację tak, aby ustawić statyczny adres IP (np. 10.100.100.3/24). Wzór wygląda tak: 

network:
  ethernets:
    enp0s3:
      dhcp4: no
      addresses: [10.100.100.3/24]
      routes:
        - to: default
          via: 10.100.100.1
      nameservers:
        addresses: [8.8.8.8, 8.8.4.4]
  version: 2
🚨 KRYTYCZNY BŁĄD (Spacje vs TAB): W plikach YAML NIE WOLNO UŻYWAĆ KLAWISZA TAB! Wciśnięcie TABa zepsuje cały plik i sieć nie wstanie. Wcięcia robimy wyłącznie spacjami (zazwyczaj po 2 spacje na każdy poziom wcięcia). Zwróć uwagę na to, jak równo ułożone są linijki w przykładzie powyżej.
# 3. Zapisz plik (Ctrl+O, Enter, Ctrl+X) # 4. Zastosuj nową konfigurację sieci sudo netplan apply # 5. Sprawdź, czy karta dostała nowy adres IP ip a

Treść zadania:

‒ serwer DNS: adres interfejsu LAN_switch na serwerze

📸 Weryfikacja Egzaminacyjna: Zrzut konfiguracji do pliku

Egzaminatorzy często proszą o zapisanie aktualnej konfiguracji sieciowej do pliku tekstowego, aby mogli ją później sprawdzić.

# Zapisz wynik komendy 'ip a' do pliku o nazwie 'konfiguracja' w swoim katalogu domowym ip a > ~/konfiguracja # Sprawdź, czy plik zawiera poprawne dane cat ~/konfiguracja
